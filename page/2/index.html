新的互联网围墙

2019年08月15日
计算机
留言

最近想让搜索引擎收录下Blog，发现事情并不是那么简单了。百度依然占居国内最大搜索市场，你会发现其搜索结果排前列的，绝大部分内容是百度自己的服务如百家号、百度百科、百度贴吧等。互联网看来是互通，但搜索引擎却将将它变成另外一个围墙了。

在百度提交网站首页URL后，一直没有消息。考虑百度信息垄断，搜索流量都引入自己服务，决定考虑另外一个搜索引擎，查询国内PC端搜索引擎市场占有率排名如下：

百度：58.67%
360搜索：25.88%
谷歌：5.26%
搜狗：4.42%
必应：4.28%

对排名第二的360一直没有好印象，几乎没用过其产品，直接PASS；谷歌一直被墙，找梯子太麻烦，有插件生成其sitemap，也不考虑了；搜狗直接无视；最后只剩下微软的必应，知道其偶尔也会被墙，决定还是试试看。

通过www.cn.bing.com/webmaster ，可以选择微软、google、facebook账号登录。

登录后，添加网站URL。注意了，千万不要使用https，否则会影响后面的验证。添加网站URL时可以顺便添加网站sitemap，Bing的 Sitemap feed(s)支持如下格式： Sitemap 、RSS 2.0、Atom 0.3、Atom 1.0 和文本文件。

接下来，共有三种验证方法：

1.将 XML 文件放在网络服务器上
2.复制 标记并在默认网页上粘贴
3.手动向 DNS 添加 CNAME 记录

一般选择第一种验证，简单与方便。记住网站URL不要使用https，否则无法通过验证。

添加sitemaps后，一般三天左右，使用“site:添加的域名”就可以搜索了。对于一些重要页面，为了增加Bing收录时效性，可以通过“提交 URL”，基本上当天提交，当天就可搜索到。

后补：由于关联熊掌ID需要手持身份证照片，没有关联之，多次提交网站首页URL后，百度仍没有收录。最近看了一则长沙的新闻，才明白咋回事，原来是第八十六条的原因。

2019年08月12日
计算机
留言

最近同时在使用Github和Coding，两者都支持Git仓库。而Git是目前世界上最先进的分布式版本控制系统，相较于SVN，分布式版本系统的最大好处之一是在本地工作完全不需要考虑远程库的存在，也就是有没有联网都可以正常工作！当有网络的时候，再把本地提交推送一下就完成同步了，非常方便，Git得到了众多IT开发平台和工作者的青睐！

Linux上使用Git，在各发行版本中非常方便。Windows上使用Git，需要从Git官网直接下载（网速慢的可以去国内镜像或者迅雷下载），然后按默认选项安装即可。我选择的是Portable版本，需要自己设置环境变量，将git的位置添加到PATH 中即可。PortableGit中有两个可执行文件：git-bash.exe和git-cmd.exe，看名字就知道怎么回事了。建议使用git-bash，可记录命令历史。关于Git的具体命令使用，这里就不再赘述了。

1. 生成密钥文件
